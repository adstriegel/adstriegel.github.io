How To Read The Papers


When reading the papers for the class, take the following approach:
  • Read the abstract
  • Think: What is the problem they are attempting to solve?
  • Skim the paper from start to finish fairly quickly noting key sections / information
  • Think: How are they solving this problem?
  • Read the paper at a reasonable pace noting anything interesting
  • Think: Did the paper solve the problem?
  • Meet with a group and discuss the paper or raise a discussion item on Campuswire

Pre-Review Notes

Before class, there will be the following materials posted:
  • A brief overview the day before about what we will be talking about with respect to lecture
  • A set of general questions that can be found in the text directly
  • A set of questions that go deeper that may not have a definitive answer but are excellent group discussion questions
Consider the following questions that can be found in the papers directly:
  • Paper 1 - Nucleus
    • What is the difference between a program and a process?
    • What is an external process?
    • What is a message queue?
    • What part handles interrupts?
  • Paper 2 - UNIX Timesharing
    • What is the difference between an image and a process?
    • What does fork do?
    • What does pipe do?
    • What does wait do?
    • Where does the shell look for executables?
    • How does the shell run a command?
    • What is a core file?
Consider the following questions in a group or by yourself (see above):
  • Paper 1 - Nucleus
    • What is the difference between the message buffer and message queue? Why does it matter?
    • What does it mean to write (a file) in Nucleus?
    • Explain hierarchy in Nucleus.
    • Are buffers necessarily in the system nucleus?
  • Paper 2 - UNIX Timesharing
    • The paper notes that locks are not needed, has that held up?
    • The paper describes fork, is that the same today as it was back then?
    • Why does the paper specifically note the primary focus is on files?
  • Both Papers
    • Why does the distinction between code and running code matter?
    • Is a parent process in UNIX afforded the same control as Nucleus affords parent processes?
